# Laravel + Vue.js SPA 開発タスク (v3)

## 概要
前回の修正後、いくつかのエラーが新たに発生し、未解決のUIの問題も残っています。以下のタスクを実行してください。

---

## 1. エラー修正

### 1.1. プロフィールページのルートエラー
右上のプロフィールアイコンから「プロフィール設定」をクリックすると、以下のエラーが発生してページが開けません。
```
Uncaught (in promise) Error: Ziggy error: route 'profile.edit' is not in the route list.
```
JavaScript側で `route('profile.edit')` を使用していますが、Ziggyのルートリストに含まれていないようです。ルートキャッシュをクリアするか、ルートが正しくフロントエンドに共有されているか確認し、修正してください。

### 1.2. アンケート結果ページの表示エラー
アンケート管理画面で「結果を見る」ボタンを押すと、画面がクラッシュし、以下のエラーがコンソールに表示されます。
```
Uncaught (in promise) TypeError: Cannot read properties of null (reading 'useContext')
at exports.useContext (react.development.js:1212:25)
```
このエラーは、Vue.jsアプリケーション内でReactベースのライブラリ (`recharts` など) を使用しようとしていることが原因と考えられます。`SurveyResults.vue` を調査し、互換性のないReact製チャートライブラリを、`vue-chartjs` や `ApexCharts` などのVue互換のライブラリに置き換えて、アンケート結果が正しくグラフ表示されるように修正してください。

---

## 2. UI/UXの改善

### 2.1. 日付・時刻の表示フォーマット統一
カレンダーやリマインダーなどで、日時が `2025-10-17T15:00:00.000000Z` のようにISO形式で表示されています。これを `2025-10-12` のような、より人間が読みやすい形式に整形してください。関連する全てのコンポーネントを修正してください。

### 2.2. 残存する黒い枠線の修正
多くのコンポーネントで枠線が黒く表示される問題がまだ残っています。
- **対象箇所:**
  - ほとんどのボタン (`Button`)
  - テキスト入力欄 (`Input`, `Textarea`)
  - チップ (`Badge`)
根本的な原因は、グローバルCSS (`app.css`) で定義されている `--border` 変数にあると考えられます。この変数の値を、より薄いグレー (例: `hsl(240 5.9% 90%)`) に変更して、アプリケーション全体の枠線の色を修正してください。
