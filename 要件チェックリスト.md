# 要件チェックリスト

## プロジェクト計画書の要件確認

このチェックリストは、プロジェクト計画書で定義された不満点が解消されているかを定期的に確認するためのものです。
新機能追加や仕様変更時には、必ずこのチェックリストを確認してください。

---

## 不満点1: 全社共有で使いにくい
### 解消策: 部署内で予定を共有できるカレンダー

**チェック項目**:
- [ ] 各部署ごとに独立したカレンダーが存在するか
- [ ] 部署カレンダーは部署メンバーのみがデフォルトで表示されるか
- [ ] 全社カレンダーと部署カレンダーが明確に区別されているか
- [ ] サイドバーで部署を切り替えてカレンダーを表示できるか
- [ ] 部署管理者/メンバーは自部署のカレンダーに簡単にアクセスできるか

**現在の実装状況**: ✅ 実装済み
- DashboardSidebar.vueで部署切り替え機能実装済み
- owner_department_idで部署カレンダーを管理

**リスク要因**:
- 権限管理の複雑化により、部署カレンダーの境界が曖昧になる
- 全社カレンダーに予定が集中し、部署カレンダーが使われなくなる

**対策**:
- カレンダー作成時に「どのカレンダーに作成するか」を明示的に選択させる
- 部署カレンダーをデフォルト表示にする（現在の部署を優先）

---

## 不満点2: 検索機能が使いづらい
### 解消策: 日付や名前、部分一致の検索機能

**チェック項目**:
- [ ] 横断検索バーで予定・共有メモ・アンケートを一括検索できるか
- [ ] 部分一致検索が機能しているか
- [ ] 日付範囲での絞り込みができるか
- [ ] 作成者名での検索ができるか
- [ ] 部署フィルターで特定部署の予定のみ表示できるか
- [ ] 検索結果が見やすく表示されているか

**現在の実装状況**: ✅ 実装済み
- GlobalSearch.vueで横断検索機能実装済み
- 部署フィルター、カテゴリフィルター実装済み

**リスク要因**:
- フィルター項目が増えすぎてUIが複雑になる
- 検索速度が遅くなる（データ量増加時）

**対策**:
- フィルターは必要最小限に抑える（現在の仕様を維持）
- データベースインデックスを適切に設定
- 検索結果の表示件数制限（ページネーション）

---

## 不満点3: 別のアプリとLINEWORKSの予定の衝突
### 解消策: アプリ内で予定を一括管理する

**チェック項目**:
- [ ] カレンダー機能で全ての予定を管理できるか
- [ ] 共有メモをカレンダーにリンクできるか
- [ ] アンケートで予定のすり合わせができるか
- [ ] 通知センターで全ての予定を一覧できるか
- [ ] 外部アプリとの連携が不要な状態になっているか

**現在の実装状況**: ✅ 実装済み
- カレンダー、共有メモ、アンケート機能実装済み
- 通知センターで一括管理可能

**リスク要因**:
- 機能が分散し、結局複数画面を行き来する必要が出る
- 外部ツール（LINEWORKS等）との併用が続く

**対策**:
- ダッシュボードで全機能にアクセスできる設計を維持
- 通知センターを強化し、全ての情報を一箇所で確認できるようにする
- 外部カレンダーのインポート機能は追加しない（一元管理を徹底）

---

## 不満点4: 紙の付箋だと必要な情報がすぐに確認できない
### 解消策: 閲覧専用タブレットで誰でも予定やメモを共有できるようにする

**チェック項目**:
- [ ] 共有メモ機能が実装されているか
- [ ] 共有メモをカレンダーにリンクできるか
- [ ] 公開範囲を設定できるか（全社/部署/カスタム）
- [ ] タブレット等のデバイスからアクセスできるか
- [ ] 閲覧専用モードまたはゲストアクセスが可能か

**現在の実装状況**: ✅ 実装済み
- 共有メモ機能実装済み
- カレンダーとのリンク機能実装済み
- 公開範囲設定実装済み

**リスク要因**:
- 閲覧専用アカウントの管理が煩雑になる
- タブレット用のUIが最適化されていない

**対策**:
- レスポンシブデザインを維持（タブレット対応）
- 「閲覧専用」ロールの追加を検討（編集権限なし）
- 大画面表示モードの追加を検討

---

## 不満点5: 明確な予定のすり合わせ方法がない
### 解消策: 誰が投票したかどうかがわかるアンケート機能

**チェック項目**:
- [ ] アンケート機能が実装されているか
- [ ] 誰が投票したかを確認できるか
- [ ] 投票結果が集計されて表示されるか
- [ ] 公開範囲を設定できるか（全社/部署/カスタム）
- [ ] アンケート結果をカレンダーに反映できるか

**現在の実装状況**: ✅ 実装済み
- アンケート機能実装済み
- 投票者表示機能実装済み
- 公開範囲設定実装済み

**リスク要因**:
- アンケート機能が使われず、結局口頭やLINEでのすり合わせに戻る
- アンケート作成が面倒で利用されない

**対策**:
- アンケートテンプレート機能の追加を検討（「日程調整」「出欠確認」等）
- カレンダーから直接アンケートを作成できる導線を強化
- アンケート結果から自動で予定を作成する機能を検討

## 定期確認スケジュール

### 月次確認（毎月末）
- 各機能が正常に動作しているか
- ユーザーからの不満が再発していないか
- 新機能追加時に既存機能が損なわれていないか

### 四半期確認（3ヶ月ごと）
- プロジェクト計画書の要件が全て満たされているか
- 新たな不満点が発生していないか
- 改善要綱の見直し

### 年次確認（1年ごと）
- プロジェクト計画書の全面見直し
- 要件の追加・削除・変更の検討

---

## 新機能追加時のチェックフロー

新機能を追加する際は、以下を確認してください：

1. **既存機能への影響確認**
   - この変更により、上記5つの不満点解消策が損なわれないか？
   - 既存のユーザー体験が悪化しないか？

2. **要件との整合性確認**
   - この機能はプロジェクト計画書の要件に沿っているか？
   - 新たな不満点を生み出さないか？

3. **シンプルさの維持**
   - 機能が複雑になりすぎていないか？
   - ユーザーが迷わず使えるか？

## 警告サイン（再発の兆候）

以下のような状況が発生した場合、要件が満たされなくなっている可能性があります：

⚠️ **警告サイン**:
- ユーザーが「どこに予定を作ればいいかわからない」と言い始める
- 検索機能が使われず、直接ファイルを探す行動が見られる
- 外部ツール（LINEWORKS等）の利用が再開される
- 共有メモが使われず、紙の付箋が復活する
- アンケート機能が使われず、口頭でのすり合わせに戻る
- 「全社カレンダーが見づらい」という不満が出る
- 部署カレンダーが使われず、全社カレンダーに集中する

これらの兆候が見られた場合は、即座に原因を調査し、改善要綱に反映してください。

---

## 改善要綱への反映ルール

このチェックリストで問題が発見された場合：

1. **改善要綱_2026年3月.md** の「今後の改善予定項目」に追加
2. 優先度を設定（高/中/低）
3. 具体的な改善策を記載
4. 実装完了後、このチェックリストを再確認

## まとめ

プロジェクト計画書の5つの不満点は、このシステムの存在意義そのものです。
これらが満たされなくなった時点で、システムの価値は失われます。

**定期的にこのチェックリストを確認し、要件が満たされ続けているかを監視してください。**
