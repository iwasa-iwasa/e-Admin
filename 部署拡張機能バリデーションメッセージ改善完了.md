## 公開範囲の選択肢

すべての機能で以下の4つの公開範囲が選択可能です：

1. **public（全社公開）**: 全社員が閲覧可能
2. **department（部署公開）**: 所属部署のメンバーのみ閲覧可能
3. **private（非公開）**: 作成者のみ閲覧可能
4. **custom（カスタム）**: 指定したユーザーのみ閲覧可能

## テスト方法

### 1. 予定作成時のバリデーション
```
1. カレンダーページで「新規予定」をクリック
2. タイトルのみ入力して「公開範囲」を選択せずに保存
3. 「公開範囲を選択してください。」と表示されることを確認
```

### 2. 共有メモ作成時のバリデーション
```
1. 共有メモページで「新規メモ」をクリック
2. タイトルのみ入力して「公開範囲」を選択せずに保存
3. 「公開範囲を選択してください。」と表示されることを確認
```

### 3. アンケート作成時のバリデーション
```
1. アンケートページで「新規アンケート」をクリック
2. タイトルと質問を入力して「公開範囲」を選択せずに保存
3. 「公開範囲を選択してください。」と表示されることを確認
```

### 4. プロフィール更新時のバリデーション
```
1. プロフィール設定ページを開く
2. 所属部署を未選択にして保存
3. 「所属部署を選択してください。」と表示されることを確認
```

## 注意事項

1. **必須項目**: `visibility_type` は必須項目として設定されています
2. **デフォルト値**: フロントエンドでデフォルト値を設定することを推奨します
3. **部署公開**: `visibility_type` が 'department' の場合、`owner_department_id` が自動的に設定されます
4. **バリデーション順序**: バリデーションは定義順に実行されます

## 今後の改善案

1. **フロントエンドでのリアルタイムバリデーション**: ユーザーが入力中にエラーを表示
2. **ツールチップの追加**: 各公開範囲の説明をツールチップで表示
3. **デフォルト値の設定**: ユーザーの所属部署に応じたデフォルト値の設定
4. **バリデーションルールの統一**: 全機能で共通のバリデーションルールを使用
