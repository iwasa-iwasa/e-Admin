### ① VueDatePicker の日付フォーマットを日本形式に統一

#### 概要

全画面の VueDatePicker で表示される日付が海外形式（`MM/DD/YYYY HH:mm`）になっている。
`YYYY/MM/DD HH:mm` の日本形式に統一する。

#### 対象ファイル（共通ラッパーなし・各ファイルを個別修正）

| ファイル | 対象箇所 |
|---|---|
| `SharedCalendar.vue` | イベント新規作成・編集フォームの日時入力 |
| `CreateNoteDialog.vue` | 共有メモ新規作成の期限入力 |
| `NoteDetailDialog.vue` | 共有メモ編集の期限入力 |
| `Notes.vue` | 共有メモ一覧のフィルター日付範囲 |
| `SurveyForm.vue` | アンケート作成・編集の回答期限入力 |
| `Surveys.vue` | アンケート一覧のフィルター日付範囲 |
| `PersonalReminders.vue` | 個人リマインダーの期限入力 |
| `Trash.vue` | ゴミ箱のフィルター日付範囲 |
| `GlobalSearch.vue` | 横断検索の日付範囲フィルター |

#### 実装方針

各ファイルの `<VueDatePicker>` コンポーネントに `:format` プロパティを追加する。

**単一日時選択（時刻あり）の場合:**

```vue
<VueDatePicker
  v-model="date"
  :format="'yyyy/MM/dd HH:mm'"
/>
```

**単一日付選択（時刻なし）の場合:**

```vue
<VueDatePicker
  v-model="date"
  :format="'yyyy/MM/dd'"
/>
```

**日付範囲選択（フィルター等）の場合は関数形式で渡す:**

```vue
<VueDatePicker
  v-model="dateRange"
  range
  :format="formatDateRange"
/>
```

```typescript
const formatSingle = (date: Date): string => {
  const y = date.getFullYear()
  const m = String(date.getMonth() + 1).padStart(2, '0')
  const d = String(date.getDate()).padStart(2, '0')
  const h = String(date.getHours()).padStart(2, '0')
  const min = String(date.getMinutes()).padStart(2, '0')
  return `${y}/${m}/${d} ${h}:${min}`
}

const formatDateRange = (dates: Date[]): string => {
  return dates.map(d => formatSingle(d)).join(' - ')
}
```

#### 注意事項

- `format` プロパティは**表示フォーマットのみ**を変更する。`v-model` に渡す値（Date オブジェクト）や API に送る値には影響しない。
- 時刻入力が不要なフィールド（日付のみ）は `HH:mm` を省略すること。
- 全9ファイルを対象にすること。修正漏れがないよう上記テーブルをチェックリストとして使用すること。

### ⑤ 共有メモ専用画面の部署フィルターが機能しない問題の修正

#### 概要

`Notes.vue` のフィルターポップアップから部署を選択しても、メモ一覧が絞り込まれず全件表示されたままになっている。

#### 現象

- フィルターで部署を選択 → 閉じる → 一覧が変わらない（全件表示のまま）
- 他のフィルター（作成者・メモの種類・タグ）は正常に動作している（要確認）

#### 調査・修正対象

**フロントエンド（`Notes.vue`）を最初に確認すること:**

1. フィルターの部署選択値が `ref` または `reactive` で保持されているか
2. フィルター実行時（ポップアップを閉じた時）に、部署フィルター値が API リクエストのパラメーターに含まれているか
3. フィルター値のキー名が API 側の受け取りキー名と一致しているか（例: `department_id` vs `department`）

**バックエンド（`NoteController.php`）を次に確認すること:**

1. `index` メソッドで `department_id`（または相当するキー）を `$request->input()` で受け取っているか
2. 受け取った値を Eloquent クエリの `where` 句に適用しているか
3. `owner_department_id` と `department_id` のカラム名の対応が正しいか

#### 修正方針

フロントで値が渡されていない場合はフロントを修正。
バックエンドでフィルタリングが行われていない場合は `NoteController.php` の `index` メソッドを修正。
両方に問題がある場合は両方修正する。

### ⑥ ユーザー管理画面（Admin.vue）の権限・部署管理機能の拡張

#### 概要

現状の `Admin.vue` では以下ができていない。これらを実装する。

1. **全社管理者への権限変更ができない** → 権限変更の選択肢に「全社管理者」が存在しない
2. **部署名の変更ができない** → 部署名の編集 UI が存在しない

#### 6-1. 全社管理者への権限変更

##### 現状

ユーザー一覧の権限変更ドロップダウンに「全社管理者」の選択肢がない。

##### 実装内容

**フロントエンド（`Admin.vue`）:**

権限変更のセレクトボックスに「全社管理者」を追加する。

```typescript
const roleOptions = [
  { value: 'company_admin', label: '全社管理者' }, // ← 追加
  { value: 'department_admin', label: '部署管理者' },
  { value: 'member', label: 'メンバー' },
]
```

表示・操作制限のルール:

- 全社管理者のみがこのドロップダウンを操作できる
- 自分自身の権限は変更不可（グレーアウトまたは非表示）
- 全社管理者が0人になる操作は不可（最低1人を必須とする）

警告表示: 「全社管理者」に変更する際は確認ダイアログを表示する。表示文は「〇〇さんを全社管理者に変更します。全社管理者はすべての部署の情報にアクセスできます。よろしいですか？」

**バックエンド（`UserController.php` または相当するコントローラー）:**

- `role` カラムへの `company_admin` の保存を許可する
- 自分自身への変更を拒否するバリデーションを追加する
- 全社管理者が最低1人残るかチェックするバリデーションを追加する

```php
$request->validate([
    'role' => ['required', 'in:company_admin,department_admin,member'],
]);

if ($user->id === auth()->id()) {
    abort(403, '自分自身の権限は変更できません');
}

if ($user->role === 'company_admin' && $request->role !== 'company_admin') {
    $adminCount = User::where('role', 'company_admin')->count();
    if ($adminCount <= 1) {
        abort(422, '全社管理者は最低1人必要です');
    }
}
```

#### 6-2. 部署名の変更機能

##### 現状

部署の一覧は表示されているが、部署名を編集する UI が存在しない。

##### 実装内容

**フロントエンド（`Admin.vue`）:**

部署一覧の各行に「編集」ボタンを追加し、インライン編集で部署名を変更できるようにする。推奨 UI はインライン編集（鉛筆アイコンをクリック → テキスト入力フィールドに切り替え → 確定ボタン）。全社管理者のみ編集ボタンを表示する。部署名は必須・重複不可。確認ダイアログは不要。

**バックエンド（`DepartmentController.php` または相当するコントローラー）:**

`update` メソッドが存在しない場合は新規作成する。

```php
public function update(Request $request, Department $department)
{
    if (auth()->user()->role !== 'company_admin') {
        abort(403);
    }
    $request->validate([
        'name' => ['required', 'string', 'max:255', Rule::unique('departments', 'name')->ignore($department->id)],
    ]);
    $department->update(['name' => $request->name]);
    return response()->json($department);
}
```

**ルーティング（`routes/api.php` または `web.php`）:**

```php
Route::put('/departments/{department}', [DepartmentController::class, 'update']);
```

### ⑦ カレンダーのホバーツールチップに所属部署名を追加

#### 概要

全社・全部署の予定を表示している場合、どの部署の予定かが区別できない。全社または全部署表示時のみ、既存のホバーツールチップ（タイトル・重要度・進捗・締め切り）に所属部署名を追加表示する。

#### 表示条件

- **表示する**: 全社表示または全部署（複数部署）表示時
- **表示しない**: 特定の1部署に絞り込んでいる場合（自明なため不要）

#### 表示内容・デザイン

既存のツールチップの先頭または末尾に「所属」行を追加する。表示内容の例: 全社 / 総務部 / 営業部

色分けルールは通知センター・横断検索バーと統一する。全社は紫色バッジ、自部署は青色バッジ、他部署はオレンジ色バッジ。

#### 実装方針

**フロントエンド（`SharedCalendar.vue` 内のツールチップ該当部分）:**

既存のツールチップコンポーネントに `ownerDepartment` プロパティを追加する。カレンダーの表示モード（全社/全部署/特定部署）を判定する既存の状態変数を参照し、全社・全部署表示時のみバッジを描画する。

```vue
<span v-if="showDepartmentBadge" :class="departmentBadgeClass(event.owner_department_id)">
  {{ event.owner_department_name ?? '全社' }}
</span>
```

```typescript
const departmentBadgeClass = (departmentId: number | null): string => {
  if (!departmentId) return 'bg-purple-100 text-purple-700'
  if (departmentId === currentUser.department_id) return 'bg-blue-100 text-blue-700'
  return 'bg-orange-100 text-orange-700'
}
```

イベントデータに `owner_department_name` が含まれていない場合は、バックエンドのイベント取得 API のレスポンスに `owner_department_name` を追加する（`EventController.php` または相当）。
