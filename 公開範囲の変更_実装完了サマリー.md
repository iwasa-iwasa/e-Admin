## 変更内容の詳細

### カレンダー予定

#### 変更前
```php
// ユーザーが公開範囲を選択
'visibility_type' => 'required|string|in:public,department,private,custom',

// 手動で設定
'visibility_type' => $data['visibility_type'] ?? 'public',
```

#### 変更後
```php
// バリデーションから削除

// カレンダーの種別から自動決定
$calendar = Calendar::findOrFail($data['calendar_id']);

if ($calendar->owner_type === 'company') {
    $visibility_type = 'public';
    $owner_department_id = null;
} elseif ($calendar->owner_type === 'department') {
    $visibility_type = 'department';
    $owner_department_id = $calendar->owner_id;
}
```

## データベースマイグレーション実行

```bash
php artisan migrate
```

このマイグレーションにより、既存の不合理なデータが自動的に修正されます：

1. **カレンダー予定**
   - `private` → `department` に変更
   - `custom` → カレンダーの種別に応じて変更

2. **共有メモ**
   - `private` → `department` に変更
   - `public`（予定に紐づかない） → `department` に変更

3. **アンケート**
   - `private` → `department` に変更

## ユーザーへの影響

### カレンダー予定
- **変更前**: 公開範囲を選択する必要があった
- **変更後**: カレンダーを選択すると自動的に公開範囲が決まる

### 共有メモ
- **変更前**: 全社公開・非公開を選択できた
- **変更後**: 部署内共有・カスタム共有のみ選択可能

### アンケート
- **変更前**: 非公開を選択できた
- **変更後**: 全社公開・部署公開・カスタムのみ選択可能

## 今後の拡張

### 全社カレンダー承認フロー
部署の予定を全社に共有したい場合：
1. 部署カレンダーで予定を作成（部署内のみ表示）
2. 全社カレンダーへの追加を申請
3. 全社管理者が承認
4. 全社カレンダーに予定がコピーされる（全社公開）

この機能により、部署カレンダーから直接全社公開する必要がなくなります。
